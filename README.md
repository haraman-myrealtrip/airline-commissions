## 사이트 구조

### 1. 파일 구성

| 파일 | 설명 |
|------|------|
| `index.html` | **실제 배포본.** HTML·CSS·JS가 모두 인라인으로 들어있는 단일 파일(약 1,165줄). Pages가 이 파일을 서빙함. |
| `commission.html` | 초기 버전으로 보이는 파일. 현재 `index.html`을 포함해 어디에서도 링크되지 않는 **고아(orphan) 파일** — 배포/사용 안 됨. |

> 별도의 빌드 과정·프레임워크·CI 없음. `index.html`을 `main`에 푸시하면 GitHub Pages가 자동 재배포(보통 1~2분).

### 2. 데이터 흐름 (핵심)

이 대시보드는 **"HTML 하드코딩 = 초기값/폴백", "구글시트 = 실제 화면에 뜨는 값"** 2단 구조입니다.

```
페이지 로드
  │
  ├─ 1) loadEmbeddedData()   → index.html 안에 하드코딩된 배열로 화면 초기화
  │        airlines[]  (커미션 요약 표/카드)
  │        historyData[] (상세 이력 표)
  │
  └─ 2) loadSheetsData()     → 구글시트를 읽어 위 값을 "덮어씀"
           대시보드_커미션 탭  → airlines[]의 각 항목을 코드 기준으로 갱신
           대시보드_히스토리 탭 → historyData[]를 통째로 교체
```

**중요**: 그래서 HTML만 수정하고 시트를 안 바꾸면, 로드 시 시트값이 이겨서 **라이브 화면은 안 바뀝니다.** 항목을 실제로 반영하려면 **HTML(코드 기준값)과 구글시트(화면 값)를 둘 다 동일하게** 맞춰야 합니다.

### 4. 읽기 vs 쓰기 (권한 구조)

| 동작 | 방식 | 필요 권한 |
|------|------|-----------|
| **시트 → 사이트 반영(읽기)** | 구글시트 공개 `gviz` 엔드포인트 | 로그인만 되면 OK, **토큰과 무관** |
| **사이트 → 시트 저장(쓰기)** | Google Sheets API v4 (`values.update` / `append`) | **관리자 OAuth 토큰(1시간)** 필요 |
| 이미지 첨부 | Google Drive API 업로드 후 공개 링크 | 관리자 토큰 |

- 인증: Google Identity Services(GSI). 허용 도메인 `ALLOWED_DOMAINS = ['myrealtrip.com','aicx.kr']`
- 관리자: `ADMIN_EMAILS` 목록(`index.html` 상단 상수)에 등록된 계정만 저장 기능 사용 가능

## 사이트 문제점 및 TODO

### [1] 원인 — "왜 토큰 만료가 반복되는지"

현재 관리자 저장 기능은 구글 로그인 시 받은 **1시간짜리 임시 권한(토큰)** 으로만 동작합니다.
1시간이 지나면 권한이 만료되는데, 자동 재발급이 없어 저장 버튼을 눌러도 아무 일도 안 일어나는 현상이 발생합니다.
참고로 **시트 → 사이트 반영 기능은 토큰과 무관**하다고 보이며, 페이지 새로고침 및 구글 캐시 때문에 반영이 늦어 보일 수 있습니다.

### [2] 단기 대응 (차주)

저장하기 직전에 토큰이 만료됐는지 확인하고, 필요하면 자동으로 재발급 후 저장하도록 하는 로직 추가가 필요하다고 보여집니다.
대부분의 경우 사용자는 재로그인 없이 계속 사용할 수 있습니다.

### [3] 장기 대응 (지속적인 사용을 위한 개선 방안)

브라우저에서 직접 구글 시트를 수정하는 구조를 없애고, **Google Apps Script를 중간 서버처럼 사용**하는 방식으로 전환을 검토해볼 수 있을 것 같습니다.
이렇게 변경하면 토큰 만료 개념 자체가 사라져, 장시간 사용해도 저장이 끊기지 않을 것으로 기대됩니다.
